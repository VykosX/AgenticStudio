# Source Grounded Research

## When To Use

- User asks for research, domain context, source-first answers, or synthesis.
- The answer depends on current external information or standards.
- Multiple sources need to become decisions, tradeoffs, and implementation notes.

## Workflow

1. Pick a mode: `domain_research`, `source_first`, or `synthesis`.
2. Prefer primary sources and official documentation when available.
3. Use targeted site search when a domain-specific source is better than broad web search.
4. Capture source URLs, dates, and the specific claim each source supports.
5. Summarize narrowly and avoid copying long source text.
6. Separate facts, inferences, tradeoffs, and recommendations.
7. Save durable findings to memory when they will guide future work.

## Preferred Tools

- `as_multi_website_search`
- `as_web_search`
- `as_web_extract`
- `as_http_request`
- `as_memory_controller`
- `as_agent_task`

## Output

- question
- sources
- findings
- tradeoffs
- recommendation
- open uncertainties
